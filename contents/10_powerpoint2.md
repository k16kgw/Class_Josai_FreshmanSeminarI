## スライド構成の確認

発表資料を作るときは，最初に全体の流れを確認する．
スライドを作りながら内容を考えると，話題がずれたり，同じ内容を繰り返したりしやすい．

### 基本構成

本授業では，次の構成を基本とする．

1. タイトル
2. 発表の目的
3. 背景または問題
4. 主張
5. 根拠
6. 具体例
7. まとめ
8. 参考文献

すべてを別々のスライドにする必要はない．
発表時間に合わせて，内容をまとめる．

### 発表時間を意識する

短い発表では，スライドを増やしすぎないことが重要である．
1枚のスライドにつき30秒から1分程度で説明すると考えると，3分発表では5枚程度が目安になる．

```{note} 演習1
その上で，スライド全体の構成を確認し，次の内容が含まれているか確認せよ．

- タイトル
- 発表の目的
- 主張
- 根拠
- 具体例
- まとめ
- 参考文献
```

## レイアウトを整える

スライドの内容が良くても，配置が乱れていると読みにくくなる．
文字，図形，画像の位置をそろえ，情報のまとまりが分かるようにする．

### 整列

複数の図形やテキストボックスは，整列機能を使ってそろえる．

- 図形またはテキストボックスを複数選択する
- 図形の書式設定タブ＞配置＞整列
- 左揃え，中央揃え，上揃えなどを選択する

### 余白

スライドの端まで文字を詰め込まない．
上下左右に余白を残すと，読みやすいスライドになる．

### フォントの統一

一つのスライド内で多くのフォントを使うと，統一感がなくなる．
基本的には，同じフォントを使い，サイズや太字で強弱をつける．

### 強調

強調したい語句は，太字や色で示す．
ただし，強調箇所が多すぎると，どこが重要なのか分からなくなる．

```{note} 演習2
第10回のPowerPointファイルについて，次の点を修正せよ．

- 各スライドのタイトル位置をそろえる
- 本文の文字サイズを読みやすくする
- 箇条書きの行頭をそろえる
- 図形や画像の位置を整える
- 色を使いすぎている場合は減らす
```

## 図で説明する

発表では，図を使うと内容を短時間で伝えやすくなる．
特に，関係，流れ，比較を説明するときに図は有効である．

### 関係を示す図

例

```text
課題管理 → 提出忘れの防止 → 学習時間の確保
```

矢印を使うと，原因と結果の関係を示しやすい．

### 比較を示す図

例

| 悪い習慣 | 良い習慣 |
| --- | --- |
| 締切を覚えておくだけ | カレンダーに記録する |
| 試験前だけ勉強する | 授業後に復習する |
| 課題を後回しにする | 早めに着手する |

表を使うと，違いを整理しやすくなる．

### 図を使うときの注意

- 図の目的を明確にする
- 文字を入れすぎない
- 矢印の向きをそろえる
- 図だけで意味が分かるようにタイトルを付ける
- 必要に応じて出典を書く

```{note} 演習3
自分の発表内容について，関係，流れ，比較のいずれかを説明する図を1つ作成せよ．

図には，短いタイトルを付けること．
```

## 発表者ノート

PowerPointには，発表者ノートを入力する機能がある．
発表者ノートには，発表時に話す内容のメモを書く．

### スライドと原稿の違い

スライドには，要点だけを書く．
発表者ノートには，口頭で説明する内容を書く．

スライドの例

```text
課題管理が重要

- 締切が授業ごとに異なる
- 提出方法も異なる
- 記録しないと忘れやすい
```

発表者ノートの例

```text
大学では授業ごとに課題の締切や提出方法が異なります．
そのため，記憶だけで課題を管理すると，提出忘れが起こりやすくなります．
私は，課題が出された時点でカレンダーに締切を記録することが重要だと考えます．
```

スライドに話す内容をすべて書くのではなく，スライドと発表者ノートを分ける．

```{note} 演習4
各スライドの発表者ノートに，発表時に話す内容を2文以上で記入せよ．

ただし，スライド本文とまったく同じ文章を繰り返さないこと．
```

## 発表練習

完成したスライドは，実際に声に出して確認する．
声に出すことで，説明しにくい箇所や，時間が長すぎる箇所に気づける．

### 確認する点

- 発表の流れは自然か
- 1枚のスライドに時間をかけすぎていないか
- 文字は読みやすいか
- 図や画像は説明に役立っているか
- 最後に結論が伝わるか

### 発表時間

本課題では，3分程度の発表を想定する．
発表時間が長すぎる場合は，内容を削る．
短すぎる場合は，根拠や具体例を補う．

---

## 提出前チェック

提出前に，次の点を確認する．

- ファイル名は指定通りか
- 1枚目にタイトル，学籍番号，氏名があるか
- スライドは5枚以上あるか
- 発表の目的が示されているか
- 主張，根拠，具体例があるか
- 図形または画像を使っているか
- 参考文献が必要な場合，最後に書いているか
- 発表者ノートを記入しているか
