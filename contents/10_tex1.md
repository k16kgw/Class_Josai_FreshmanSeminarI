## TeXとLaTeX

- **TeX**：文章や数式を一定の規則に従って配置し，読みやすい文書を作るための**組版システム**
  - Donald E. Knuth によって1978年から開発されている言語処理系
  - 日本語圏では「てふ」，英語圏では「てっく」と呼ばれることが多い
  - バージョン3以降は更新されるごとに円周率の桁数を増やしている（2021年2月に出た最新版は ver.3.141592653）
    ![Knuth](./figs/10/KnuthAtOpenContentAlliance.jpg)
    「Donald E. Knuth」Wikipedia, URL:https://commons.wikimedia.org/wiki/File:KnuthAtOpenContentAlliance.jpg#/media/%E3%83%95%E3%82%A1%E3%82%A4%E3%83%AB:KnuthAtOpenContentAlliance.jpg より
- **LaTeX**：**TeX上で動き**，見出し・箇条書き・数式・図表などを扱いやすくした文書作成システム（マクロパッケージ）
  - Leslie Lamport によって開発された
    ![Lamport](./figs/10/Leslie_Lamport.jpg)
    「Leslie Lamport」Wikipedia, URL: https://commons.wikimedia.org/wiki/File:Leslie_Lamport.jpg#/media/File:Leslie_Lamport.jpg より

※ 厳密にTeXとLaTeXは違うカテゴリであるが，区別せずに単に「TeX」と呼ばれることが多い

参考：日本語 TeX 開発コミュニティ「LaTeX」TeX wiki, URL: https://texwiki.texjp.org/LaTeX （閲覧日：2026.6.20）

### Wordとの違い

| 項目 | Word | TeX |
| --- | --- | --- |
| 編集方法 | 画面上の文書を直接編集する | 文章と命令を記述する |
| 文書の作成 | 編集画面が完成形に近い | ソースファイルからPDFを作る |
| 数式 | 数式入力機能を利用する | 数式を命令として入力する |
| レイアウト | 利用者が個別に調整する | 文書の構造に基づいて自動調整される |

TeXでは見た目を一つずつ調整するのではなく，「これは見出し」「これは数式」のように<span style="color:red">文書の構造</span>を記述する．
そのため，レポート・論文・卒業論文・書籍などの数式を多く含む文書や構造がはっきりした文書の作成に適している．

### 命令

通常の**文章**と文書の構造や組版方法を指定する**命令**をソースファイルに記述する．

例）次のコードを考える．
```latex
\section{節のタイトル}
```

- `\section`：見出しを作る**命令**
- `{` と `}`：引数の範囲を示す区切り
- `節のタイトル`：`\section` 命令に渡す**引数**

### 基本用語

| 用語 | 一般的な意味 |
| --- | --- |
| TeXソースファイル | 文章とLaTeXの命令が記述された `.tex` ファイル |
| 命令（コマンド） | `\section` のように，LaTeXに行わせる処理を指定する`\`から始まる記述 |
| 引数 | 命令に渡す値や文字列．必須の引数は通常 `{ }` で囲む |
| TeXエンジン | TeXソースファイルを読み取り，組版処理を行うプログラム |
| タイプセット | TeXソースファイルをTeXエンジンで処理し，組版結果を作ること |
| 出力 | タイプセットによって作られるPDFなどの処理結果 |
| ログ | タイプセット中の処理内容や警告，エラーを記録したもの |
| エラー | ソースの誤りなどにより，処理を正常に継続できない状態 |

今回は，TeXShopから**LuaLaTeX**を選択してTeXソースファイルをタイプセットする．
LuaLaTeXは，LuaTeXエンジンを用いてLaTeX文書をタイプセットするためのプログラムである．
TeXShopは，TeXソースファイルを編集し，TeXエンジンを呼び出し，組版結果をプレビューするためのグラフィカルユーザインターフェース（GUI）である．

### 文書作成の流れ

TeXによる文書作成やプログラミングでは，ソースファイルを編集し，対応する処理系に与えて，得られた結果を確認する．
この手順は，今後プログラミング言語を利用する際にも共通する．
最初から完成したものを一度で作ろうとせず，少し入力して結果を確認してから次へ進むことが重要である．

TeXでは，次の手順を繰り返して文書を作成する．

1. **編集する**：`.tex` ファイルに文章とLaTeXの命令を書く
2. **保存する**：入力した内容をファイルに記録する
3. **タイプセットする**：TeXShopからLuaLaTeXを呼び出してソースファイルを処理する
4. **出力を確認する**：作成されたPDFが意図した表示になっているか確かめる
5. **修正する**：誤りや変更したい箇所をソースファイル上で直す
6. **再度タイプセットする**：保存してから再びLuaLaTeXで処理する

文章とLaTeXの命令を記述した `.tex` ファイルは，一般に**TeXソースファイル**または単に**ソースファイル**と呼ばれる．
TeXソースファイルをTeXエンジンで処理して組版結果を作ることを**タイプセット**という．
**コンパイル**という語が使われることもある．

```text
ソースファイルを編集
        ↓
      保存
        ↓
LuaLaTeXでタイプセット
        ↓
 PDFまたはエラーを確認
        ↓
ソースファイルを修正
```

PDFを直接編集するのではなく，必ずソースファイルを修正してから再度タイプセットする．
この「編集・保存・処理・出力の確認・修正」の繰り返しは，今後プログラミング言語を学ぶ際にも基本となる．

## 最初のTeX文書

`main.tex` に次の内容を入力する．

```latex
% !TEX TS-program = lualatex
\documentclass[a4paper,12pt]{ltjsarticle}
\usepackage{amsmath,amssymb}

\title{TeXによる文書作成}
\author{学籍番号　氏名}
\date{\today}

\begin{document}

\maketitle

\section{はじめに}

本稿では，TeXを用いて数式を含む文書を作成する．

\section{数式}

2次関数 $f(x)=x^2+2x+1$ を考える．

\section{おわりに}

TeXでは，文章と命令を組み合わせて文書を作成する．

\end{document}
```

1行目は，TeXShopでLuaLaTeXを使うことを指定している．
削除せず，ファイルの先頭に置くこと．

### TeXソースを入力するときの注意

最初は，通常の文章とTeXの命令を区別して入力する必要がある．

- 日本語の文章は通常どおり入力する
- `\section` や `\begin` などの命令は，半角英数字で正確に入力する
- 命令の途中に空白を入れない
- 大文字と小文字を区別する
- `{` と `}`，`\begin` と `\end` など，対になる記号や命令を両方書く
- 意味が分からない命令を，文書の骨組みから不用意に削除しない

たとえば，次の2行は同じ命令ではない．

```latex
\section{はじめに}
\Section{はじめに}
```

正しい命令は小文字で始まる `\section` である．
命令のつづりや大文字・小文字が違うと，LuaLaTeXは別の命令として解釈し，エラーになる場合がある．

### コメント

行の途中に `%` を書くと，その右側は処理時に無視される．
これを**コメント**という．

```latex
% この行はPDFには表示されない
\section{はじめに} % 最初の見出し
```

コメントは，TeXソースに説明や作業メモを残すために使える．
ただし，今回の1行目にある `% !TEX TS-program = lualatex` は，TeXShopが使用するプログラムを選ぶために読み取る特別なコメントなので削除しないこと．

### タイプセット

タイプセットとは，TeXソースファイルをTeXエンジンで処理し，PDFなどの組版結果を作成することである．
初回は次の手順を一つずつ確認する．

1. **入力位置を確認する**
   文章と命令がソースウィンドウに入力されていることを確認する．
2. **ファイルを保存する**
   `command + S` を押し，`main.tex` を保存する．
3. **使用するプログラムを確認する**
   ソースウィンドウ上部の選択欄に「LuaLaTeX」と表示されていることを確認する．
4. **タイプセットする**
   「タイプセット」ボタンをクリックする．`command + T` でも実行できる．
5. **処理結果を待つ**
   LuaLaTeXがソースファイルを先頭から読み，命令を解釈して文書を組み立てる．
6. **PDFを確認する**
   正常に終了するとプレビューウィンドウが開き，作成されたPDFが表示される．
7. **ソースとPDFを見比べる**
   タイトル，氏名，見出し，文章，数式が意図したとおりに表示されているか確認する．

タイプセット中には，LuaLaTeXが概ね次の処理を行っている．

1. ソースファイルを先頭から読む
2. `\documentclass` や `\usepackage` に従って文書の設定を準備する
3. `\begin{document}` 以降の文章と命令を解釈する
4. 見出し，文章，数式などの配置を計算する
5. `main.pdf` を作成する
6. 処理内容をコンソールウィンドウに表示する

```{note} 保存とタイプセットの違い
**保存**は，ソースウィンドウの内容を `main.tex` に記録する操作である．

**タイプセット**は，保存された `main.tex` をLuaLaTeXで処理して `main.pdf` を作る操作である．

ソースファイルを変更しただけではPDFは更新されない．
変更後は「保存→タイプセット→PDF確認」の順で操作すること．
```

### 正常にタイプセットできた場合

正常にタイプセットできると，作業用フォルダには次のようなファイルが作られる．

| ファイル | 内容 |
| --- | --- |
| `main.tex` | 自分が編集するソースファイル |
| `main.pdf` | タイプセットによって作られた提出用文書 |
| `main.log` | LuaLaTeXが行った処理の記録 |
| `main.aux` | 文書を作成する途中で使われる補助ファイル |

基本的に学生が編集するのは `main.tex` だけである．
`main.log` や `main.aux` をWordなどで編集する必要はない．

### エラーが出た場合

命令のつづりや記号の対応に誤りがあると，PDFが作られず，コンソールウィンドウにエラーメッセージが表示されることがある．
これはコンピュータの故障ではなく，TeXエンジンが「どの箇所を規則どおりに解釈できなかったか」を知らせている状態である．

エラーが出たときは，次の順で対応する．

1. コンソールウィンドウで最初に表示されたエラーを確認する
2. エラーに表示された行番号を確認する
3. ソースウィンドウで，その行と直前の数行を見る
4. 最後に入力・変更した箇所を元の例と比較する
5. 一か所修正する
6. 保存してから，再びタイプセットする

一度に複数箇所を変更すると，どの修正によって直ったのか分からなくなる．
最初は一か所ずつ修正し，そのたびに組版結果を確認すること．

### TeX文書の基本構造

TeX文書は，大きく**プリアンブル**と**本文**に分かれる．

```latex
\documentclass[a4paper,12pt]{ltjsarticle}
\usepackage{amsmath,amssymb}
```

`\begin{document}` より前をプリアンブルという．
文書の種類，用紙，文字サイズ，使用する機能などを指定する．

```latex
\begin{document}

ここに本文を書く．

\end{document}
```

`\begin{document}` から `\end{document}` までが本文である．
PDFに表示したい文章や数式は，原則としてこの間に書く．

### 命令と波括弧

TeXの命令は，通常 `\` から始まる．
命令の対象は `{` と `}` で囲むことが多い．

```latex
\title{文書のタイトル}
\section{見出し}
\textbf{太字にする文字}
```

`\section{見出し}` は，見出しの番号，文字の大きさ，前後の余白を自動的に整える．

## TeXで数式を書く

TeXでは，数式として扱う範囲を明示する必要がある．

### 文中の数式

文章の途中に置く数式は `$` と `$` で囲む．

```latex
$a$ と $b$ を実数とする．
関数 $f(x)=ax+b$ のグラフは直線である．
```

文中の数式は，記号や短い式を文章の一部として示す場合に使う．

### 独立行の数式

重要な式や長い式は，`\[` と `\]` で囲んで独立した行に置く．

```latex
2次方程式 $ax^2+bx+c=0$ の解は，次のように表される．
\[
  x=\frac{-b\pm\sqrt{b^2-4ac}}{2a}
\]
ただし，$a\neq 0$ とする．
```

式だけを置くのではなく，式の前後で記号の意味や条件を文章で説明すること．

### 基本的な数式の入力

| 表したいもの | TeXでの入力 |
| --- | --- |
| べき乗 | `x^2` |
| 添字 | `a_i` |
| 複数文字の添字 | `a_{ij}` |
| 分数 | `\frac{a}{b}` |
| 平方根 | `\sqrt{x}` |
| プラスマイナス | `\pm` |
| 等しくない | `\neq` |
| 総和 | `\sum_{i=1}^{n} a_i` |
| 積分 | `\int_0^1 f(x)\,dx` |
| ギリシャ文字 | `\alpha`，`\beta`，`\theta` |

上付き・下付きに複数の文字を書く場合は，その範囲を `{` と `}` で囲む．

```latex
x^{10}
a_{ij}
\sum_{i=1}^{n}
```

### 行列

行列は `pmatrix` 環境を使って入力できる．
`&` は列の区切り，`\\` は行の終わりを表す．

```latex
\[
  A=
  \begin{pmatrix}
    1 & 2 \\
    3 & 4
  \end{pmatrix}
\]
```

行列の積を用いると，連立1次方程式を次のように表せる．

```latex
\[
  \begin{pmatrix}
    2 & 1 \\
    1 & 3
  \end{pmatrix}
  \begin{pmatrix}
    x \\
    y
  \end{pmatrix}
  =
  \begin{pmatrix}
    5 \\
    7
  \end{pmatrix}
\]
```

## エラーへの対応

TeXでは，命令や記号に誤りがあるとタイプセットが停止することがある．
エラーが出たときは，ソースファイルが壊れたと考えるのではなく，入力を一つずつ確認する．

### よくある入力ミス

- `{` と `}` の数が合っていない
- `$` を閉じ忘れている
- 命令の先頭の `\` を忘れている
- `\begin{...}` と `\end{...}` が対応していない
- 行列で列数や `&` の数が合っていない
- 半角で入力すべき記号を全角で入力している

### 修正の手順

1. エラーが表示された行番号とその前後を確認する
2. 最後に入力・変更した箇所を確認する
3. `{ }`，`$`，`\begin` と `\end` の対応を確認する
4. 一度に多くの箇所を直さず，一か所ずつ修正してタイプセットする
5. 原因が分からない場合は，追加した部分を一時的に削除して確認する

エラーが示された行より前に原因があることもある．
表示された行だけでなく，その直前の数行も確認すること．

```{note} 演習4
演習3で作成したソースの `\frac{n(n+1)}{2}` から最後の `}` を一つ削除してタイプセットし，エラーが出ることを確認せよ．

確認後は削除した `}` を元に戻し，再びPDFを作成せよ．
```

## まとめ

- TeXでは，ソースファイルに文章と命令を入力し，タイプセットしてPDFを作成する．
- ソースファイルを用いる作業では，「編集→保存→処理→出力の確認→修正」を繰り返す．
- 文書の設定はプリアンブルに書き，本文は `\begin{document}` と `\end{document}` の間に書く．
- 文中の数式は `$...$`，独立行の数式は `\[...\]` で入力する．
- 数式だけを置かず，記号の意味や式の役割を文章で説明する．
- エラーが出たときは，直前の変更箇所と記号の対応を確認する．
