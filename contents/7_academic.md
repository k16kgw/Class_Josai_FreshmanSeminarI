## 段落を作る

段落は，文章の意味のまとまりである．
一つの段落には，一つの中心的な話題を書く．

### 段落の基本構造

段落は，次のような構造にすると読みやすい．

1. 話題を示す文
2. 説明や具体例
3. まとめや次の話題への接続

たとえば，次のように書ける．

```text
大学生活では，課題の管理が重要である．
高校までと比べて，大学では科目ごとに提出方法や締切が異なることが多い．
そのため，課題が出されたらすぐに締切と提出方法を確認し，カレンダーなどに記録しておく必要がある．
```

この段落では，最初の文が中心的な話題を示している．
次に，その理由を説明し，最後に具体的な行動へつなげている．

### 段落を分ける基準

次のような場合には，段落を分ける．

- 話題が変わる
- 理由から具体例へ移る
- 具体例から結論へ移る
- 反対意見や別の観点を述べる
- 時間や場面が変わる

段落を分けずに長く書くと，読み手は文章の構造を把握しにくくなる．

````{note} 演習3
課題を忘れずに提出するためには，締切を管理する必要がある．
```
````

## 文章全体の構成

長い文章を書くときは，いきなり本文を書き始めるのではなく，構成を考えてから書く．
構成がないまま書くと，途中で話題がずれたり，結論が曖昧になったりする．

### 基本構成

大学の説明文や短いレポートでは，次の構成を意識するとよい．

1. はじめに
2. 本文
3. おわりに

### はじめに

「はじめに」では，文章の目的や扱う話題を示す．
読み手に対して，これから何について述べるのかを知らせる部分である．

```text
本稿では，大学生活における課題管理の重要性について述べる．
```

### 本文

「本文」では，主張の理由や具体例を書く．
必要に応じて，複数の段落に分ける．

```text
大学では，科目ごとに課題の提出方法や期限が異なる．
そのため，課題を記憶だけで管理するのは難しい．
```

### おわりに

「おわりに」では，文章全体のまとめを書く．
新しい話題を急に出すのではなく，本文で述べた内容を受けて結論を示す．

```text
以上のように，大学生活では課題の締切と提出方法を記録し，計画的に取り組むことが重要である．
```

## 主張と根拠

大学で書く文章では，「そう思った」だけではなく，理由や根拠を示す必要がある．

### 主張

主張とは，文章で自分が伝えたい中心的な考えである．

```text
大学生活では，課題の締切管理が重要である．
```

### 根拠

根拠とは，主張を支える理由や事実である．

```text
大学では，科目ごとに提出期限や提出方法が異なるためである．
```

### 具体例

具体例は，主張や根拠を読み手に分かりやすくするために使う．

```text
たとえば，ある授業ではWebClassで提出し，別の授業ではWordファイルをTeamsに提出することがある．
```

主張，根拠，具体例を組み合わせると，文章は説得力をもつ．

```text
大学生活では，課題の締切管理が重要である．
大学では，科目ごとに提出期限や提出方法が異なるためである．
たとえば，ある授業ではWebClassで提出し，別の授業ではWordファイルをTeamsに提出することがある．
そのため，課題が出された時点で締切と提出方法を記録する必要がある．
```

## レポートとは何か

大学で書くレポートは，単なる感想文ではない．
自分の考えを，根拠に基づいて説明する文章である．

### 感想文との違い

感想文では，自分が感じたことや印象を中心に書く．
一方，レポートでは，あるテーマについて調べたり考えたりした結果を，読み手に分かるように整理して書く．

感想文の例

```text
私は大学生活では課題管理が大切だと思いました．
なぜなら，課題が多くて大変だからです．
```

レポートらしい文の例

```text
大学生活では，課題の締切管理が重要である．
大学では科目ごとに提出方法や期限が異なるため，記憶だけで課題を管理することは難しい．
そのため，課題が出された時点で締切と提出方法を記録する必要がある．
```

レポートでは，「思った」だけで終わらせず，理由や具体例を示すことが重要である．

### レポートで求められること

レポートでは，次の点が評価される．

- 課題の指示を理解しているか
- テーマが明確であるか
- 主張が明確であるか
- 主張を支える根拠があるか
- 文章の構成が分かりやすいか
- 引用や参考文献が適切に示されているか
- 指定された形式を守っているか

大学のレポートでは，内容だけでなく形式も重要である．
どれだけよい内容を書いていても，提出条件や引用のルールを守っていなければ評価が下がる場合がある．

## レポートの基本構成

短いレポートでは，次の構成を基本とする．

1. 表紙
2. はじめに
3. 本文
4. おわりに
5. 参考文献

### 表紙

表紙には，レポートを特定するための情報を書く．

- 課題名
- 授業名
- 担当教員名
- 所属
- 学籍番号
- 氏名
- 提出日

表紙は，読み手が「誰が，どの授業で，何の課題として提出した文書か」を確認するために必要である．

### はじめに

「はじめに」では，レポートで扱うテーマと目的を示す．
読み手に対して，これから何について述べるのかを知らせる部分である．

例

```text
本レポートでは，大学生活における課題管理の重要性について述べる．
大学では複数の授業で同時に課題が出されるため，計画的に課題を管理する必要がある．
```

「はじめに」で詳しい説明を書きすぎる必要はない．
レポート全体の方向を示すことが目的である．

### 本文

本文では，主張を支える理由や具体例を述べる．
必要に応じて，複数の見出しや段落に分ける．

本文で書く内容の例

- 現状や背景
- 問題点
- 理由
- 具体例
- 調べた情報
- 自分の考察

本文では，段落ごとに話題を分ける．
一つの段落に複数の話題を詰め込みすぎると，読み手が内容を整理しにくくなる．

### おわりに

「おわりに」では，レポート全体のまとめを書く．
本文で述べた内容を受けて，結論を示す．

例

```text
以上のように，大学生活では課題の締切と提出方法を記録し，計画的に取り組むことが重要である．
課題管理を習慣にすることで，提出忘れを防ぎ，学習時間を確保しやすくなる．
```

「おわりに」で，本文にない新しい話題を急に出してはいけない．
結論は，本文で説明した内容から自然に導く．

### 参考文献

参考文献には，レポートを書くために参照した本，Webページ，論文などを記録する．
参考文献を書くことで，読み手は情報の出典を確認できる．

参考文献は，レポートの最後にまとめて書く．

## テーマ，問い，主張

レポートを書くときは，いきなり本文を書き始めない．
まず，テーマ，問い，主張を整理する．

### テーマ

テーマとは，レポートで扱う大きな話題である．

例

```text
大学生活における課題管理
```

テーマだけでは，まだ書く内容が広すぎる．
そこで，テーマをもとに問いを作る．

### 問い

問いとは，レポートで考える中心的な質問である．

例

```text
なぜ大学生活では課題管理が重要なのか．
```

問いがあると，レポートで何を説明すればよいかが明確になる．

### 主張

主張とは，問いに対する自分の答えである．

例

```text
大学生活では，科目ごとに提出期限や提出方法が異なるため，課題管理が重要である．
```

テーマ，問い，主張の関係は次のように整理できる．

| 項目 | 例 |
| --- | --- |
| テーマ | 大学生活における課題管理 |
| 問い | なぜ大学生活では課題管理が重要なのか |
| 主張 | 科目ごとに提出期限や提出方法が異なるため，課題管理が重要である |

````{note} 演習1
次のテーマについて，問いと主張をそれぞれ1つずつ作成せよ．

```text
大学生の時間管理
```
````

## 根拠を示す

レポートでは，主張を述べるだけでは不十分である．
主張を支える根拠を示す必要がある．

### 根拠の種類

根拠には，次のようなものがある．

- 授業で学んだ内容
- 教科書や配布資料の説明
- 信頼できるWebページの情報
- データや調査結果
- 具体的な事例
- 自分の経験を整理したもの

自分の経験も根拠になる場合がある．
ただし，自分の経験だけでは一般的な説明として弱いことがあるため，必要に応じて資料やデータで補う．

### 主張と根拠の対応

主張と根拠は対応していなければならない．

悪い例

```text
大学生活では課題管理が重要である．
私は朝起きるのが苦手である．
```

この例では，2文目が1文目の根拠になっていない．

改善例

```text
大学生活では課題管理が重要である．
大学では科目ごとに提出期限が異なり，複数の課題を同時に進める必要があるためである．
```

根拠を書くときは，「その理由で本当に主張を支えられているか」を確認する．

````{note} 演習2
次の主張を支える根拠を2つ書け．

```text
大学生にはタイピング練習が必要である．
```
````

## 引用と参考文献

レポートでは，本やWebページなどから得た情報を使うことがある．
その場合は，どこから得た情報なのかを示す必要がある．

### 引用とは

引用とは，他人の文章，図，データ，考え方などを，自分の文章の中で利用することである．
引用する場合は，引用した部分と自分の文章を区別し，出典を示す．

引用が必要なものの例

- 本や論文の文章
- Webページの説明
- 統計データ
- 図や画像
- 他人の考え方や定義

### 直接引用

直接引用は，他人の文章をそのまま使う方法である．
短い文章をそのまま使う場合は，引用符を付け，出典を示す．

例

```text
著者は「文章は読み手を意識して書く必要がある」と述べている（山田，2024）．
```

直接引用では，文字を勝手に変えてはいけない．
長く引用しすぎると，自分の文章ではなくなるため，必要最小限にする．

### 間接引用

間接引用は，他人の説明や考え方を，自分の言葉で要約して使う方法である．
自分の言葉で書き直していても，考え方の出所を示す必要がある．

例

```text
山田（2024）は，文章を書く際には読み手を意識することが重要であると説明している．
```

レポートでは，直接引用よりも間接引用を使うことが多い．
ただし，自分の言葉に直しても，出典を書かないと盗用になる場合がある．

### Webページを参考文献に書く

Webページを参考文献に書く場合は，次の情報をできるだけ記録する．

- 著者または組織名
- ページ名
- URL
- 閲覧日

例

```text
城西大学「図書館」https://libopac.josai.ac.jp/ （閲覧日：2026年6月1日）
```

Webページは内容が変更されることがあるため，閲覧日を書く．

### 本を参考文献に書く

本を参考文献に書く場合は，次の情報を記録する．

- 著者名
- 書名
- 出版社
- 出版年

例

```text
藤岡敦『手を動かしてまなぶ 微分積分』裳華房，2019年．
```

授業や分野によって参考文献の書き方は異なる．
指定がある場合は，必ずその指定に従う．

````{note} 演習3

次の情報を含めること．

- 組織名またはページ作成者
- ページ名
- URL
- 閲覧日
````

## 盗用を避ける

盗用とは，他人の文章，考え，データ，画像などを，自分のもののように使うことである．
大学では，盗用は重大な不正行為である．

### 盗用になる例

- Webページの文章をコピーして，出典を書かずに提出する
- 友人のレポートを一部だけ書き換えて提出する
- 本の説明をほぼそのまま使い，参考文献を書かない
- 生成AIが作成した文章を，自分で書いた文章としてそのまま提出する
- 画像や図を出典なしで使う

### 盗用を避ける方法

盗用を避けるために，次の点を守る．

- 他人の文章を使う場合は引用として示す
- 調べた情報の出典をメモしておく
- コピーした文章と自分の文章を混ぜない
- 自分の言葉で説明し直す
- 参考文献を最後にまとめる
- 生成AIを使った場合は，使い方を記録する

調べながら書くときは，どの情報をどこから得たのか分からなくなりやすい．
資料を読んだ時点で，URLや書名をメモしておくことが重要である．

## 生成AIの利用記録

生成AIは，レポート作成の補助として利用できる．
ただし，AIの出力をそのまま提出してはいけない．
また，授業や課題で利用ルールが指定されている場合は，必ずその指示に従う．

### 利用してよい例

- テーマを考えるための相談
- 文章の誤字脱字の確認
- 主張と根拠の対応の点検
- 段落構成の改善案の確認
- 参考文献の書き方の確認

### 注意が必要な例

- 課題文を入力し，全文を作成させる
- AIが示した情報を確認せずに使う
- 出典のない説明を事実として書く
- 学籍番号や氏名などの個人情報を入力する
- 自分が理解していない文章を提出する

### 利用記録の例

生成AIを使った場合は，レポートの最後に次のように記録する．

```text
生成AIの利用記録
使用ツール：ChatGPT
使用目的：文章の誤字脱字の確認，主張と根拠の対応の確認
使用日：2026年6月1日
反映内容：指摘された誤字を修正し，本文第2段落に具体例を追加した．
```

AIを使っていない場合は，次のように書く．

```text
生成AIの利用記録
本レポートでは生成AIを使用していない．
```

利用記録を書くことで，どの部分を自分で考え，どの部分でAIを補助として使ったのかを説明できる．

## レポートを書く手順

レポートは，次の手順で作成するとよい．

1. 課題文を読む
2. 提出条件を確認する
3. テーマを決める
4. 問いを作る
5. 主張を決める
6. 根拠となる情報を集める
7. 構成を作る
8. 本文を書く
9. 引用と参考文献を確認する
10. 推敲と校正を行う
11. ファイル名と提出先を確認する

### 課題文を読む

課題文には，レポートで守るべき条件が書かれている．
特に次の点を確認する．

- テーマ
- 字数
- ファイル形式
- ファイル名
- 提出先
- 提出期限
- 参考文献の必要性
- 生成AIの利用ルール

課題文を読まずに書き始めると，内容がよくても条件を満たさないレポートになることがある．

### 構成メモを作る

本文を書く前に，簡単な構成メモを作る．

例

```text
テーマ：大学生活における課題管理
問い：なぜ課題管理が重要なのか
主張：提出期限と提出方法が科目ごとに異なるため，課題管理が重要である

はじめに：大学生活では複数の課題を同時に扱う
本文1：大学の課題は提出方法や期限が異なる
本文2：締切を記録することで提出忘れを防げる
おわりに：課題管理は学習時間を確保するためにも重要である
```

構成メモがあると，本文を書いている途中で話題がずれにくくなる．

````{note} 演習4
次のテーマについて，構成メモを作成せよ．

```text
大学生にとってタイピング練習が必要な理由
```

構成メモには，次の項目を含めること．

- テーマ
- 問い
- 主張
- はじめにで書くこと
- 本文で書くこと
- おわりにで書くこと
````

## まとめ

- レポートは，感想だけでなく，根拠に基づいて自分の考えを説明する文章である．
- レポートでは，テーマ，問い，主張，根拠を整理してから書く．
- 基本構成は，表紙，はじめに，本文，おわりに，参考文献である．
- 引用や参考文献を書くことで，情報の出典を読み手に示せる．
- 他人の文章，考え，データ，画像を自分のもののように使うと盗用になる．
- 生成AIは補助として使えるが，出力をそのまま提出してはいけない．
- 生成AIを使った場合は，使用目的や反映内容を記録する．
